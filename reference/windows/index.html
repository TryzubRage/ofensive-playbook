Typical attack flow on a Windows Docker host

  1. Land inside a Linux container (from an earlier Linux exploit).
  2. Fingerprint the container (/.dockerenv, caps, sockets, ports).
  3. Spot a Docker API on 192.168.65.x:2375 (Docker Desktop's internal bridge).
  4. Create a Privileged: true container with Binds: /mnt/host/c/:/mnt/windows.
  5. Start, exec, and walk C:\Users\Administrator\Desktop.
