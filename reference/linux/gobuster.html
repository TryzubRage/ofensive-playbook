gobuster vhost / dir / dns

Directory, file, DNS and virtual-host brute-forcer. Most useful here for vhost discovery against a single IP serving multiple domains.

Key modes

ModePurpose
dirDirectory / file brute-forcing
vhostVirtual-host header brute-forcing
dnsDNS subdomain brute-forcing
fuzzGeneric fuzzing with FUZZ marker

1. Virtual-host enumeration (the one I actually used)

gobuster vhost -u http://monitorsfour.htb \
  -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt \
  --append-domain
--append-domain appends the base domain automatically — otherwise gobuster would hit raw words like cacti instead of cacti.monitorsfour.htb.

Result on MonitorsFour:

cacti.monitorsfour.htb  Status: 302 [Size: 0] [--> /cacti]

2. Directory mode (alternative to ffuf)

gobuster dir -u http://monitorsfour.htb \
  -w /usr/share/wordlists/seclists/Discovery/Web-Content/big.txt \
  -x php,html,txt -t 50

3. DNS subdomain mode

gobuster dns -d monitorsfour.htb \
  -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt \
  -t 50
