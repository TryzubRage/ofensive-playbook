ffuf web content / API / vhost fuzzing

Fast web fuzzer written in Go. Used for directory, file, parameter, API and virtual-host discovery.

Key flags (quick reference)

FlagMeaning
-uTarget URL. Use FUZZ as the injection point.
-wWordlist to iterate.
-HCustom HTTP header (e.g., Host: foo).
-eExtensions to append (e.g., .php,.bak,.env).
-fs / -fwFilter responses by body size / word count.
-mcMatch specific status codes.
-tThread count (default 40).
-cColorized output.

1. Basic directory fuzzing

Scan top-level paths with a generic wordlist:

ffuf -u http://monitorsfour.htb/FUZZ \
  -w /usr/share/wordlists/seclists/Discovery/Web-Content/big.txt

2. REST API endpoint discovery

Use an API-focused wordlist against the API prefix:

ffuf -u http://monitorsfour.htb/api/v1/FUZZ \
  -w /usr/share/wordlists/seclists/Discovery/Web-Content/api/api-endpoints-res.txt
Endpoints like /user, /users, /auth, /logout often surface here.

3. Fuzzing behind a virtual host

Pass the Host header to reach vhosts like cacti.monitorsfour.htb:

ffuf -u "http://cacti.monitorsfour.htb/cacti/FUZZ" \
  -H "Host: cacti.monitorsfour.htb" \
  -w /usr/share/wordlists/dirb/common.txt \
  -fs 0 -t 50 -c

4. Deep fuzz with extensions + response filters

Use a larger list and append extensions to hunt for backups / config leaks:

ffuf -u "http://cacti.monitorsfour.htb/cacti/FUZZ" \
  -H "Host: cacti.monitorsfour.htb" \
  -w /usr/share/wordlists/seclists/Discovery/Web-Content/DirBuster-2007_directory-list-2.3-big.txt \
  -e .php,.txt,.bak,.old,.sql,.env \
  -c -t 50 -fw 1,604
Why -fw 1,604? Every false-positive response on this app had a word count of either 1 or 604. Filtering them out leaves only real hits — that's how cacti.sql showed up.

Target-flow example (MonitorsFour)

  1. Web fuzz root → discover /api/v1/
  2. API fuzz → list of user endpoints
  3. Gobuster vhost → cacti.monitorsfour.htb
  4. ffuf with extensions → find /cacti/cacti.sql
  5. Parse dump → credentials
